# WHY — North Star (Immutable)

## Purpose

`index.html` ("AI가 바꾸는 일하는 방법 — 비개발자를 위한 AI 세미나") 을:

1. **선형 내러티브** — 개념이 점진적으로 확장 (왕복 없음)
2. **시각적 가시성** — 글씨 125% 체감, 회색 일변도 → 포인트 색상 은은하게
3. **톤 담백화** — "증거/대단/짜잔" 류 자기암시·자기자랑 제거
4. **콘텐츠 정확성** — 사실 왜곡 없이 프레이밍만 재구성

## Success Criteria

- [ ] 챗봇 → 에이전트 → 멀티에이전트 → 분야확산 → 인프라병목 → 역할변화 → 현실제약 (왕복 없는 선형)
- [ ] 전 슬라이드 Chrome 검수 통과
- [ ] 폰트 가독성 체감 125% 향상, 회색 텍스트 가시성 개선
- [ ] "증거/대단/짜잔" 표현 제거, 자기자랑 톤 제거
- [ ] 메모리·인프라 섹션 "병목" 프레이밍 + 추론 시장 메모리 명시
- [ ] 하이퍼스케일러 슬라이드: 구체 숫자(5대/680B) 빼고 치킨게임/과소투자 공포 내러티브
- [ ] Demo — Claude Code 영상 링크 검증/교체

## Non-Negotiable Principles

1. **내용 왜곡 금지** — 기존 팩트(인용·수치·출처) 보존, 프레이밍만 재구성
2. **청중 우선** — 비개발자 임원이 사전지식 없이 선형 이해 가능
3. **점진적 확장** — 한 슬라이드는 직전 개념을 확장·심화
4. **은은한 디자인** — 기존 미니멀 흑/흰 톤 + 포인트 색상(`#60a5fa`/`#34d399`/`#fbbf24`/`#fca5a5`) 유지
5. **청크 I/O** — Read/Write 800줄 이내, Edit 단위 작업
6. **Chrome QA 필수** — 눈으로 확인한 것만 완료

## Anti-Patterns

- 한 번에 800줄 이상 Read/Write
- 사실 변경 (수치·인용·출처 임의 수정)
- 톤 톤다운 명목으로 정보 손실
- 디자인 시스템 깨기 (Figtree/Noto 폰트, 미니멀 흑/회 톤)
- 개념을 중간에 되돌리기
